02 Embedding vectors

Each token is mapped to a high-dimensional vector — a list of numbers that encodes meaning based on how the token appeared during training.

›

An embedding turns a token ID like 1432 into a vector like [0.21, -0.84, 0.03, ...] of length 768 to 12,288 depending on model size. Words with similar meanings end up close together in this space.

This is the model's internal dictionary. Before any attention happens, every token becomes a position in a geometric space — and that geometry captures relationships like "king – man + woman ≈ queen".

03 Positional Encoding order

Transformers have no built-in sense of order. Positional encodings inject the position of each token into its embedding so the model knows what came first.

›

Without this, "dog bites man" and "man bites dog" would look identical. The model adds a unique positional signal to each embedding before any processing begins.

Modern models like LLaMA use RoPE (Rotary Position Embedding), which encodes relative distances between tokens rather than absolute positions — making them better at generalizing to long contexts.

04 Self-Attention the core

Every token looks at every other token and decides how much to "attend" to it. This is how the model understands context, reference, and meaning.

›

For each token, the model computes three vectors: a Query (what am I looking for?), a Key (what do I contain?), and a Value (what should I pass forward?).

Attention scores are calculated as the dot product of Q and K, scaled and softmaxed into a probability distribution. The output is a weighted sum of all Values — letting "it" in "the trophy didn't fit in the suitcase because it was too big" correctly point to "trophy".

This happens in parallel across many heads — each learning different relationships (syntax, coreference, semantics).

05 Feed-Forward Layer per-token

After attention mixes token information across the sequence, a feed-forward network processes each token independently — acting as the model's "memory" for facts.

›

Each transformer block has a two-layer MLP applied identically to every token position. Research suggests these layers store factual associations — like "Paris is the capital of France".

They use non-linear activations (GeLU, SwiGLU) to create complex, high-dimensional representations that the next attention layer can then re-mix.

06 Stacking Layers depth

One attention + feed-forward block isn't enough. Models stack 12 to 96+ of these layers, each refining the representation until the final layer is ready to predict.

›

Early layers learn syntax and surface patterns. Middle layers capture semantics and relationships. Later layers handle task-specific reasoning. GPT-4 is estimated to have ~96 layers.

Between layers, residual connections and layer normalization keep gradients stable during training and prevent representations from collapsing.

07 Sampling & Output generation

The final layer outputs a probability distribution over the entire vocabulary. One token is chosen, appended to the input, and the whole process repeats.

›

The logits (raw scores) are passed through a softmax to get probabilities. Then a sampling strategy picks the next token: greedy (always highest), top-k, top-p (nucleus), or temperature scaling.

This is why LLMs are autoregressive — they generate one token at a time, each output becoming part of the next input, until an [EOS] token is produced or a length limit is reached.
